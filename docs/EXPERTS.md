# Drawhaus Expert Panel

## How To Use This Panel
- Bring a concrete decision, tradeoff, or problem.
- Ask for input from all experts or selected names.
- Expect disagreement by design; synthesize into one action.
- Default output format: recommendation, risks, fallback plan.

## Panel Operating Rules
- Disagree openly, decide clearly, and document why.
- Use "must/should/could" to rank recommendations.
- For major decisions, require:
  - one chosen option,
  - one rejected option with reason,
  - one rollback path.
- Timebox strategy debates; move to experiments quickly.

## Suggested Prompts
- "Panel, evaluate these 3 options for sharing permissions."
- "Security + Architect: review this API and socket event for auth gaps."
- "Product + Growth: which ICP should we target first and why?"
- "DevOps: define production readiness for our next milestone."
- "UX + Architect: simplify this flow without reducing capability."
